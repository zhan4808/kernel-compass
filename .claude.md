# kernel-compass — guidance for Claude

## What this repo is

Counter-grounded GPU kernel optimization for **transformer-style batched GEMMs**, especially **MLA reconstruction** shapes. The feedback signal is **hardware counters** (Nsight Compute when `ncu` is available; **CUPTI-based estimation** otherwise), not latency alone. Goal: avoid recommending weight quantization when weights are **L2-resident** and DRAM is not the bottleneck (the “cache-barrier” story).

Companion context: [cache-barrier](https://github.com/zhan4808/cache-barrier) (paper / empirical L2 boundary). This repo includes **`cache-barrier/` as a git submodule** (run `git submodule update --init --recursive` after clone). LaTeX draft and GPU data checklist live in `paper/`.

## Precision claims (do not overstate)

- **Baseline:** BF16 (default in `optimizer.loop --precision`).
- **FP8 in tree:** **weight-only W8A16** (FP8 weights, FP16 activations / Triton compute). **Not** native H100 FP8 **W8A8** tensor-core inference (cuBLASLt / Transformer Engine). Say “FP8 weight-only (W8A16)” in user-facing text; reserve “native FP8 inference” for a future W8A8 path verified with NCU.
- **INT4:** W4A16 Triton with dequant overhead; can be compute-bound on small shapes.

`PRECISION_TIERS` in `optimizer/loop.py` marks `fp8.hw_native: False` for the current implementation.

## Classifier + loop behavior (high level)

- **`l2_bound`:** low DRAM %, low SM %, high L2 hit → **do not enumerate** FP8/INT4 in `enumerate_configs` (quantization is counter-indicated).
- **`memory_bound`:** high DRAM % → try FP8 weight-only W8A16, then INT4; `verify()` requires useful latency win and guards SM saturation.
- Heuristic thresholds in `classify_one` are tuned for H100-like behavior; document if changing for A100.

## Commands (GPU machine)

```bash
pip install -r requirements.txt
python tests/test_validation.py              # timing + shapes
python tests/test_validation.py --ncu        # counters + classifier (needs GPU; NCU if permitted)
python -m optimizer.loop --demo
python -m optimizer.loop --compare
python -m optimizer.loop --grid 128 1 128 512 --precision bf16
python -m optimizer.loop --grid 128 1 128 4096 --precision bf16
python -m optimizer.loop --llm 128 1 128 4096 --precision bf16 --heuristic
```
